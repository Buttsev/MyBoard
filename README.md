# Mini-Trello (Kanban Board)

## Цель проекта

Учебно-практическое full-stack приложение для понимания:

- архитектуры клиент-серверных приложений
- управления состоянием на фронтенде
- проектирования API
- работы с реляционной БД
- аутентификации и авторизации

Фокус проекта — **понимание принципов и причин архитектурных решений**, а не скорость разработки и копипаст готовых решений.

---

## Функциональность (MVP)

### 1. Пользователи и авторизация
- Регистрация пользователя
- Логин
- JWT-аутентификация

### 2. Доски
- Список досок пользователя
- Создание доски
- Удаление доски
- Пользователь является владельцем своих досок

### 3. Колонки (внутри доски)
- Создание колонки
- Переименование колонки
- Удаление колонки
- Сортировка колонок (position)

### 4. Карточки (внутри колонок)
- Создание карточки
- Редактирование карточки
- Удаление карточки
- Перемещение карточки:
  - между колонками
  - внутри колонки
- Поддержка порядка (position)

### 5. Права доступа
- Только владелец доски имеет доступ к управлению её содержимым

## Приоритеты проекта

- Учебная ценность
- Корректная архитектура
- Ясное понимание:
  - почему выбрано именно это решение
  - какие есть альтернативы
  - какие ошибки типичны для новичков

Проект не про «быстро собрать», а про **осознанно спроектировать**.
